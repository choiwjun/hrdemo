> **프로젝트명:** 팀메이트
> **버전:** v1.0
> **작성일:** 2025-01-28
> **상태:** MVP 기획 완료

---

## 1. 문제 정의

### 1.1 현재 상황
소규모 회사(10~30명)에서 일하는 사람들은 매일 여러 도구를 오가며 업무를 처리합니다:
- 이메일 확인 → Gmail 열기
- 팀 메시지 확인 → Slack 열기
- 일정 확인 → 캘린더 앱 열기
- 출퇴근 기록 → 별도 시스템 또는 엑셀

### 1.2 핵심 문제
**"여러 툴을 따로 쓰느라 확인하기 힘들다"**

- 창 전환 피로: 하루에도 수십 번 앱을 오감
- 정보 분산: 중요한 정보가 어느 앱에 있는지 기억해야 함
- 비용 부담: 기존 그룹웨어는 복잡하고 비쌈
- 복잡한 UI: 메뉴가 너무 많아서 뭐가 어딨는지 모르겠음

### 1.3 문제의 영향
- 업무 집중력 저하
- 중요한 메시지/메일 누락 위험
- 신규 입사자 온보딩 시간 증가

## 2. 제품 비전 및 목표

### 2.1 비전
> "여러 툴을 오가는 피로감을 줄여준다"

팀메이트는 소규모 팀이 **하나의 화면에서** 메일, 메시지, 일정, 근태를 모두 확인할 수 있는 **가벼운 그룹웨어**입니다.

### 2.2 목표

| 구분 | 목표 | 측정 방법 |
|------|------|----------|
| 노스스타 지표 | 한 팀이 2주 이상 꾸준히 사용 | 일일 로그인 기록 |
| 입력 지표 1 | 일일 활성 사용자(DAU) 확보 | 로그인 수 |
| 입력 지표 2 | 통합 메일함 일일 확인율 80% | 메일함 접근 로그 |
| 입력 지표 3 | 출퇴근 기록 완료율 90% | 근태 기록 수 |

### 2.3 성공 기준 (MVP)
- [ ] 한 팀(10명 이상)이 2주간 매일 사용
- [ ] "툴 전환이 줄었다"는 정성적 피드백 수집
- [ ] 치명적 버그 없이 안정적 운영

| **직무** | 마케팅 담당자 |
| **회사** | 15명 규모 IT 스타트업 |

| **불편함** | "아침마다 Gmail, Slack, 캘린더 다 열어야 해요. 뭐 하나 확인하려면 창을 찾느라 시간 낭비..." |
| **원하는 것** | "한 화면에서 오늘 뭐 해야 하는지 딱 보고 싶어요" |

| 항목 | 내용 |
|------|------|

| **직무** | 개발팀 리드 |

## 5. 기능 요구사항

### 5.1 MVP 범위 (FEAT-1: 통합 메일함)

#### 5.1.1 메시지 목록
- Gmail과 Slack 메시지를 통합된 리스트로 표시
- 서비스별 필터 (전체 / Gmail만 / Slack만)
- 받은편지함 뷰: mention, cc, bcc, all
- 보낸편지함, 임시저장, 스팸, 휴지통 뷰
- 페이지네이션 (한 페이지 20개)
- 읽음/안읽음 상태 표시

#### 5.1.2 메시지 상세
- 스레드(대화) 형태로 메시지 표시
- 발신자, 수신자, 날짜, 내용 표시
- 첨부파일 목록 표시

#### 5.1.3 메시지 작업
- 답장, 전달
- 삭제 (휴지통으로 이동)
- 읽음/안읽음 토글
- 라벨 적용/제거

#### 5.1.4 검색
- 키워드 검색
- 검색 결과 리스트 표시

#### 5.1.5 라벨 관리
- 라벨 생성, 수정, 삭제
- 라벨 색상 지정
- 라벨별 필터링

### 5.2 MVP 이후 범위

#### Phase 2: 근태 + 캘린더
- FEAT-2: 출퇴근/근태 관리
- FEAT-3: 팀 캘린더/일정

#### Phase 3: 확장 기능
- 채팅 (실시간 팀 소통)
- 게시판/공지
- 멤버 관리

#### Phase 4: 외부 연동
- Gmail API 실제 연동
- Slack API 실제 연동
- Google Calendar 연동

#### Phase 5: AI 기능
- AI 메일 자동 요약
- AI 메일 분류 추천

## 6. 비기능 요구사항

### 6.1 성능
- 페이지 초기 로딩: 3초 이내
- 메시지 목록 로딩: 1초 이내
- 검색 결과 응답: 2초 이내

### 6.2 사용성
- 미니멀 & 깔끔한 UI (노션, 리니어 참고)
- 직관적인 메뉴 구조 (뎁스 최소화)
- 격식 있고 정중한 시스템 메시지

### 6.3 접근성
- 반응형 웹 (PC 최적화, 모바일 대응)
- 키보드 탐색 지원
- 적절한 색상 대비

### 6.4 보안
- 사용자 인증 (이메일/비밀번호)
- 세션 관리
- HTTPS 통신

## 9. 실험-학습 루프

### MVP 검증 계획

```
[실험]
→ 10명 이상 팀에게 2주간 팀메이트 사용 요청

[관측]
→ DAU, 메일함 접근율, 출퇴근 기록률 측정
→ 정성 피드백 수집 (인터뷰/설문)

[학습]
→ "어떤 기능을 가장 많이 쓰는가?"
→ "어떤 기능이 없어서 불편한가?"
→ "다시 이전 도구로 돌아가고 싶은가?"

[다음 가설]
→ 학습 결과에 따라 Phase 2 우선순위 조정
```
